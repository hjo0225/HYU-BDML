# ADR-0002: RAG 패널 선정에서 연령 필터 제거

## Status

Accepted (2026-04-15)

## Context

초기 RAG 패널 선정은 `target_customer`에서 연령대를 추출(예: "20대 직장인")하여 패널 풀을 1차 필터링한 뒤, 클러스터 다양성·주제 관련성으로 N명을 뽑았다.

문제:

- 보유 패널 500명의 연령 분포가 30~40대에 편중되어 있음.
- 20대 대상 서비스의 경우 연령 필터 적용 시 가용 패널이 극단적으로 적어져 클러스터 다양성이 무너짐.
- 일부 케이스는 N명을 채우지 못해 회의가 빈약해졌다.

## Decision

**연령 필터를 완전히 제거**한다. 전체 500명 풀에서 다음 두 차원만으로 N명을 선정한다:

- 클러스터 다양성: 0.7 가중치
- 주제 관련성 (`avg_embedding` ↔ 합성 쿼리 임베딩 코사인 유사도): 0.3 가중치

연령 등 인구통계가 정말 중요한 케이스라면, `prompts/panel_query.py`의 합성 쿼리에 자연어로 반영되어 임베딩 거리에 자연스럽게 녹아든다.

## Consequences

**긍정적**

- 모든 타겟 고객 시나리오에서 안정적으로 N명 확보.
- 클러스터 다양성이 보장되어 회의가 다양한 관점을 가짐.

**부정적**

- 연령 분포가 의도와 다를 수 있음. 사용자가 명시적으로 "20대 ○○"를 원해도 30~40대 패널이 섞일 수 있음.
- 향후 패널 풀을 균형 있게 확장하면 재검토 필요.
