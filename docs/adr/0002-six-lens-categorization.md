# ADR-0002 — 6-Lens 카테고리 분할 채택

## Status

Accepted (2026-05-07)

## Context

Twin-2K-500 의 234문항을 에이전트가 참조할 수 있도록 어떻게 카테고리화할지 결정해야 했다. archive(BDML-Lab) 의 `twin_memory_builder.py` 는 ~32 개 카테고리(`values_environment`, `decision_risk`, `social_trust` ...) 로 나누어 임베딩했다. 그러나 다음 한계가 있었다.

1. **세분화 과다.** 32 카테고리는 1:1 매핑이 가능하지만, 사용자가 "이 페르소나는 어떤 사람이야?" 라고 물을 때 카테고리를 한눈에 그룹핑하기 어렵다. 대시보드 시각화에서 32축 레이더는 가독성 없음.
2. **이론적 근거 약함.** 32 카테고리는 설문 도구(척도) 단위로 만들어졌다. 행동경제학·심리학 연구자가 봤을 때 "어떤 큰 주제를 다룬다"가 명확하지 않다.
3. **평가 지표(V1/V5)와의 연결 모호.** V5 반사실적 평가에서 "L6 시간 지향이 변하면 답변 Δ 가 어떻게 달라지나"를 측정하려면, 시간 관련 지표가 묶여 있어야 한다.

후보:

- **A. 32 카테고리 유지** (archive 방식)
- **B. 6-Lens (L1~L6) + 정성 그룹** (사용자 §5.1 제안)
- **C. Big5 / OCEAN 으로 단순화** (학계 표준)

## Decision

**B. 6-Lens (L1~L6) + 정성 + Ability + Demographics 의 9개 그룹** 채택.

- L1 경제적 합리성
- L2 의사결정 스타일
- L3 동기 (조절초점)
- L4 사회적 영향
- L5 가치 (자기/타인지향)
- L6 시간 지향
- Qualitative (자기 서술 + 게임 사고)
- Ability (금융이해 + 수리)
- Demographics (인구통계)

선택 이유:

1. **이론적 명확성.** 6 Lens 는 행동경제학·소비자심리 연구의 빅 카테고리에 직접 매핑된다 (위험·결정·동기·사회·가치·시간). 사용자(스타트업 PO)가 카테고리 의미를 즉시 이해할 수 있다.
2. **시각화 적합.** 6축 또는 5축(Identity/Logic 분리 시 V1~V5) 레이더가 한 화면에 깔끔.
3. **평가 지표(V5)와 연결.** L6 시간 지향이 묶여 있으므로, "할인율 변화 시나리오" CF 를 던지면 L6 sensitivity 가 변할 거라고 직접 예측 가능.
4. **정성·Ability 분리 보존.** 자유응답 원문은 Hybrid Persona Prompt 의 "원문 가이드"로 별도 사용. 능력치(CRT, 금융이해)는 페르소나 추론 능력의 상한선을 결정하므로 별도 그룹으로.

대안 기각 사유:

- ❌ A. 32 카테고리는 retrieval 에서는 유리하지만, 페르소나 정체성을 사람이 이해하기 어려움. 32→6 그룹핑 메타데이터를 두면 양쪽을 절충 가능 — Phase 2 에서 검토.
- ❌ C. Big5(OCEAN) 만으로는 행동경제학 핵심(위험회피·할인율) 을 표현 못 함. 미적합.

## Consequences

### 긍정적

- 대시보드 V1~V5 와 6-Lens 의 매핑이 명확. 사용자가 "이 에이전트는 V5(시간) 점수가 낮으니 L6 데이터가 부족해서 그런 거구나" 같은 추론 가능.
- Hybrid Persona Prompt 의 [수치 기반 가이드] 섹션이 6 그룹 단위로 깔끔하게 떨어진다.
- 향후 사용자 자체 Survey(Phase 6)도 같은 6-Lens 로 매핑해서 Twin/Survey 에이전트의 이질성 차단.

### 부정적

- 234문항 중 6-Lens 어디에도 명확히 안 들어가는 항목은 임의 분류 필요. 모호한 매핑은 [`6-LENS_MAPPING.md`](../6-LENS_MAPPING.md) 의 [매핑 의사결정 로그] 에 사유 기록.
- archive 의 32 카테고리 메모리 데이터(`panel_memories.source='twin2k500'` 50명 × ~32) 와 직접 호환 안 됨. 신규 적재 (`agents.source_type='twin'` 30명) 은 처음부터 6-Lens 기반. archive 데이터 통합 여부는 ADR 별도 결정.

### Open Question

- **L3 vs L5 경계.** 미니멀리즘은 L3(동기)에도 L5(가치)에도 들어갈 수 있다. 잠정: 측정 도구가 동기 이론 기반이므로 L3 분류. Phase 2 PDF 분석 시 재확인.
- **Big5 어디에?** L6 성실성은 시간 지향과 상관 있어서 L6 으로. 외향성/우호성 등 다른 Big5 차원은 본 6-Lens 에 직접 매핑 안 됨 — `scratch.big5` 별도 필드로 보존.
